# Notifications Module

Message routing and formatting for alerts and digests.

## Overview

```
notifications/
├── base.py      # Priority enum, AlertMessage, NotificationChannel ABC
├── console.py   # ConsoleChannel - logs to stdout/loguru
├── manager.py   # NotificationManager - routes by priority
└── digest.py    # Daily/weekly digest formatters
```

## Core Types

### Priority

```python
from investment_monitor.notifications import Priority

Priority.HIGH    # Send immediately
Priority.MEDIUM  # Include in daily digest
Priority.LOW     # Weekly digest or log only
```

### AlertMessage

```python
from investment_monitor.notifications import AlertMessage, Priority

msg = AlertMessage(
    title="AAPL dropped 5%",
    body="Apple fell from $185 to $175.75 today",
    ticker="AAPL",
    alert_type="price_drop",
    priority=Priority.HIGH,
    url="https://finance.yahoo.com/quote/AAPL"
)

# String representation
print(str(msg))  # "[AAPL] AAPL dropped 5%: Apple fell from..."

# Formatting helpers
msg.format_short()  # Brief version
msg.format_full()   # Detailed version
```

## Console Channel

Logs alerts using loguru with priority-based log levels.

```python
from investment_monitor.notifications import ConsoleChannel, AlertMessage, Priority

channel = ConsoleChannel()

# Send single alert
await channel.send(AlertMessage(
    title="Test",
    body="Test body",
    alert_type="test",
    priority=Priority.HIGH
))
# Logs at WARNING level (HIGH priority)

# Send digest
await channel.send_digest([msg1, msg2, msg3])
# Logs formatted digest at INFO level
```

**Log level mapping:**
| Priority | Log Level |
|----------|-----------|
| HIGH | WARNING |
| MEDIUM | INFO |
| LOW | DEBUG |

## Notification Manager

Routes messages based on priority.

```python
from investment_monitor.notifications import NotificationManager, ConsoleChannel, Priority

# Initialize with channels
manager = NotificationManager([ConsoleChannel()])

# HIGH priority - sent immediately
await manager.notify(AlertMessage(..., priority=Priority.HIGH))

# MEDIUM priority - queued for digest
await manager.notify(AlertMessage(..., priority=Priority.MEDIUM))

# LOW priority - debug log only
await manager.notify(AlertMessage(..., priority=Priority.LOW))

# Check queue
print(manager.get_pending_count())  # Number of MEDIUM messages queued

# Send digest (clears queue)
await manager.send_digest()
```

### Multiple Channels

```python
# When you add Slack/email
manager = NotificationManager([
    ConsoleChannel(),
    SlackChannel(webhook_url),
    EmailChannel(api_key),
```

### Email Channel

```python
# notifications/email.py
from sendgrid import SendGridAPIClient
from sendgrid.helpers.mail import Mail
from .base import NotificationChannel, AlertMessage

class EmailChannel(NotificationChannel):
    name = "email"

    def __init__(self, api_key: str, from_email: str, to_email: str):
        self.client = SendGridAPIClient(api_key)
        self.from_email = from_email
        self.to_email = to_email

    async def send(self, message: AlertMessage) -> bool:
        mail = Mail(
            from_email=self.from_email,
            to_emails=self.to_email,
            subject=f"[Investment Alert] {message.title}",
            plain_text_content=message.format_full()
        )
        try:
            response = self.client.send(mail)
            return response.status_code in (200, 202)
        except Exception as e:
            logger.error(f"Email send failed: {e}")
            return False

    async def send_digest(self, messages: list[AlertMessage]) -> bool:
        plain, html = format_daily_digest(messages)
        mail = Mail(
            from_email=self.from_email,
            to_emails=self.to_email,
            subject="Investment Monitor Daily Digest",
            plain_text_content=plain,
            html_content=html
        )
        response = self.client.send(mail)
        return response.status_code in (200, 202)
```

### Register New Channel

```python
# In main.py
from investment_monitor.notifications.slack import SlackChannel
from investment_monitor.notifications.email import EmailChannel

channels = [ConsoleChannel()]

if settings.slack_webhook_url:
    channels.append(SlackChannel(settings.slack_webhook_url))

if settings.sendgrid_api_key:
    channels.append(EmailChannel(
        settings.sendgrid_api_key,
        "alerts@yourdomain.com",
        "you@email.com"
    ))

manager = NotificationManager(channels)
```

## Troubleshooting

### Messages Not Appearing

1. Check log level:
   ```python
   # LOW priority uses DEBUG level
   # Make sure DEBUG logging is enabled
   from investment_monitor.logging_config import setup_logging
   setup_logging(log_level="DEBUG")
   ```

2. Check channel list:
   ```python
   print(manager.channels)  # Should have ConsoleChannel
   ```

### Digest Not Sending

1. Check pending count:
   ```python
   print(manager.get_pending_count())  # Should be > 0
   ```

2. Ensure `send_digest()` is called:
   ```python
   await manager.send_digest()  # Clears queue after sending
   ```

### HTML Rendering Issues

The HTML output uses inline styles for email compatibility. Test with:
- Gmail
- Outlook
- Apple Mail

If styling breaks, simplify `_format_html_digest()` in `digest.py`.
